Polymarket CLOB API Documentation

Base Information

REST Host: https://clob.polymarket.com

WSS Host: wss://ws-subscriptions-clob.polymarket.com

1. REST API

GET /book

Retrieves the order book summary for a specific token.

Query Parameters

ParameterTypeRequiredDescriptiontoken_idstringYesThe unique identifier for the token (Asset ID).

Response Schema (200 OK)

JSON


{

  "market": "string",          // Market condition identifier

  "asset_id": "string",        // Asset identifier

  "timestamp": "string",       // ISO 8601 date-time

  "hash": "string",            // Hash of the order book state

  "bids": [                    // Array of bid levels

    { "price": "string", "size": "string" }

  ],

  "asks": [                    // Array of ask levels

    { "price": "string", "size": "string" }

  ],

  "min_order_size": "string",  // Example: "0.001"

  "tick_size": "string",       // Example: "0.01"

  "neg_risk": "boolean"        // Whether negative risk is enabled

}

2. Websocket (WSS) Overview

Connection & Authentication

To subscribe, send a JSON message immediately upon opening the connection.

User Channel: Requires Authentication.

Market Channel: Public (No Authentication required).

Subscription Message Structure

JSON


{

  "type": "string",        // "USER" or "MARKET"

  "assets_ids": ["string"],// Array of token IDs (Required for MARKET channel)

  "markets": ["string"],   // Array of condition IDs (Required for USER channel)

  "auth": {                // Required only for USER channel

    "apiKey": "string",

    "secret": "string",

    "passphrase": "string"

  }

}

Dynamic Subscription (Market Channel)

To subscribe/unsubscribe to assets after connection:

JSON


{

  "assets_ids": ["string"],

  "operation": "subscribe" // or "unsubscribe"

}

3. WSS Channel: user (Authenticated)

This channel pushes updates on orders and trades specific to the authenticated API key.

Message Type: trade

Emitted when a market order is matched or a limit order is included in a trade. Updates occur on status changes (MINED, CONFIRMED, FAILED).

JSON


{

  "event_type": "trade",

  "type": "TRADE",

  "asset_id": "string",       // Token ID

  "id": "string",             // Trade ID

  "market": "string",         // Market Condition ID

  "outcome": "string",        // e.g., "YES"

  "price": "string",

  "size": "string",

  "side": "string",           // "BUY" or "SELL"

  "status": "string",         // "MATCHED", "MINED", etc.

  "timestamp": "string",      // Unix timestamp

  "matchtime": "string",

  "owner": "string",          // API key of event owner

  "trade_owner": "string",    // API key of trade owner

  "taker_order_id": "string",

  "maker_orders": [

    {

      "asset_id": "string",

      "order_id": "string",

      "owner": "string",

      "outcome": "string",

      "price": "string",

      "matched_amount": "string"

    }

  ]

}

Message Type: order

Emitted on order placement (PLACEMENT), update (UPDATE), or cancellation (CANCELLATION).

JSON


{

  "event_type": "order",

  "type": "string",           // "PLACEMENT", "UPDATE", "CANCELLATION"

  "id": "string",             // Order ID

  "asset_id": "string",

  "market": "string",

  "price": "string",

  "original_size": "string",

  "size_matched": "string",   // Size matched so far

  "side": "string",           // "BUY" or "SELL"

  "outcome": "string",

  "owner": "string",

  "order_owner": "string",

  "timestamp": "string",

  "associate_trades": ["string"] // IDs of trades this order is included in

}

4. WSS Channel: market (Public)

This channel pushes Level 2 price data and public trade information.

Message Type: book

Emitted upon subscription (snapshot) or when a trade affects the book.

JSON


{

  "event_type": "book",

  "asset_id": "string",

  "market": "string",

  "timestamp": "string",      // Unix timestamp (ms)

  "hash": "string",

  "bids": [

    { "price": "string", "size": "string" }

  ],

  "asks": [

    { "price": "string", "size": "string" }

  ]

}

Message Type: price_change

Note: Schema update scheduled for Sept 15, 2025.

Emitted when a new order is placed or cancelled.

JSON


{

  "event_type": "price_change",

  "market": "string",

  "timestamp": "string",

  "price_changes": [

    {

      "asset_id": "string",

      "price": "string",      // Price level affected

      "size": "string",       // New aggregate size at level

      "side": "string",       // "BUY" or "SELL"

      "hash": "string",

      "best_bid": "string",

      "best_ask": "string"

    }

  ]

}

Message Type: tick_size_change

Emitted when the market's minimum tick size changes (e.g., price moves > 0.96 or < 0.04).

JSON


{

  "event_type": "tick_size_change", // Note: Source text ambiguous, using value from example

  "asset_id": "string",

  "market": "string",

  "old_tick_size": "string",

  "new_tick_size": "string",

  "timestamp": "string"

}

Message Type: last_trade_price

Emitted when a maker and taker order match.

JSON


{

  "event_type": "last_trade_price",

  "asset_id": "string",

  "market": "string",

  "price": "string",

  "size": "string",

  "side": "string",

  "timestamp": "string",

  "fee_rate_bps": "string"

}

5. Implementation Reference (Python)

Header Generation for Auth

Python


from py_clob_client.client import ClobClient


host = "https://clob.polymarket.com"

key = "YOUR_PRIVATE_KEY"

chain_id = 137# Choose signature type based on login method (1=Email/Magic, 2=Browser Wallet)

client = ClobClient(host, key=key, chain_id=chain_id, signature_type=1)

api_creds = client.derive_api_key() # Returns: {"apiKey": "", "secret": "", "passphrase": ""}

WSS Connection Logic

URL: wss://ws-subscriptions-clob.polymarket.com/ws/market (or /user)

Ping: Send "PING" every 10 seconds.

User Auth Payload:

JSON


{

    "markets": ["condition_id_..."], 

    "type": "user", 

    "auth": {"apiKey": "...", "secret": "...", "passphrase": "..."}

}

Market Sub Payload:

JSON


{

    "assets_ids": ["token_id_..."], 

    "type": "market"

}

## 6. Client Public Methods (No Auth Required)

These methods operate without user credentials. Use for market discovery and price data.

### Initialization (Python)
```python
from py_clob_client.client import ClobClient
client = ClobClient(host="[https://clob.polymarket.com](https://clob.polymarket.com)", chain_id=137)

```

### 6.1 System & Metadata

| Method | Returns | Description |
| --- | --- | --- |
| `getOk()` | `Promise<any>` | Health check. |
| `getServerTime()` | `Promise<number>` | Current server unix timestamp (seconds). |
| `getFeeRateBps(tokenID)` | `Promise<number>` | Fee rate in basis points. |
| `getTickSize(tokenID)` | `Promise<string>` | "0.1", "0.01", "0.001", or "0.0001". |
| `getNegRisk(tokenID)` | `Promise<boolean>` | Returns true if market uses negative risk. |

### 6.2 Market Discovery

**Shared Interfaces:**

```typescript
interface MarketToken {
  outcome: string; price: number; token_id: string; winner: boolean;
}

interface Market {
  condition_id: string; question: string; market_slug: string;
  active: boolean; closed: boolean; archived: boolean;
  end_date_iso: string; game_start_time: string;
  min_order_size: number; minimum_tick_size: number;
  tokens: MarketToken[];
  rewards: { max_spread: number; min_size: number; rates: any };
  // ... (other standard metadata fields)
}

interface SimplifiedMarket {
  condition_id: string; active: boolean; closed: boolean;
  tokens: { outcome: string; price: number; token_id: string }[];
}

```

**Methods:**
| Method | Returns | Notes |
| :--- | :--- | :--- |
| `getMarket(conditionId)` | `Promise<Market>` | Get single market details. |
| `getMarkets()` | `Promise<PaginationPayload<Market>>` | Paginated list of markets. |
| `getSimplifiedMarkets()` | `Promise<PaginationPayload<SimplifiedMarket>>` | Lighter payload for fast loading. |
| `getSamplingMarkets()` | `Promise<PaginationPayload<Market>>` | Sampling variant. |

### 6.3 Order Book & Pricing

**Shared Enums:**

* `Side`: "BUY", "SELL"
* `OrderType`: "GTC", "FOK", "GTD", "FAK"

**Methods:**

* **`getOrderBook(tokenID)`**
Returns `OrderBookSummary` (see REST GET `/book` schema).
* **`calculateMarketPrice(tokenID, side, amount, orderType)`**
Returns `Promise<number>` representing the calculated execution price.
* **`getPrice(tokenID, side)`**
Returns `{ price: string }`.
* **`getMidpoint(tokenID)`**
Returns `{ mid: string }` (Average of best bid/ask).
* **`getSpread(tokenID)`**
Returns `{ spread: string }` (Difference between best ask/bid).

**Batch Methods:**

* `getOrderBooks(params: {token_id, side}[])` -> `OrderBookSummary[]`
* `getPrices(params)` -> `{ [tokenId]: { BUY?: string, SELL?: string } }`
* `getMidpoints(params)` -> `{ [tokenId]: string }`
* `getSpreads(params)` -> `{ [tokenId]: string }`

### 6.4 Historical Data

* **`getPricesHistory(params)`**
**Params:**
```typescript
{
  market: string; // tokenID
  interval: "max" | "1w" | "1d" | "6h" | "1h";
  startTs?: number; endTs?: number; fidelity?: number;
}

```


**Returns:** `Array<{ t: number, p: number }>` (Timestamp, Price)

### 6.5 Trades

* **`getLastTradePrice(tokenID)`**
Returns `{ price: string, side: string }`.
* **`getMarketTradesEvents(conditionID)`**
Returns list of recent trade events for the market.
**Response Item Schema:**
```typescript
{
  event_type: string;
  market: { condition_id: string; asset_id: string; question: string; slug: string };
  side: Side; size: string; price: string; outcome: string;
  transaction_hash: string; timestamp: string;
  user: { address: string; username: string; ... };
}

```
